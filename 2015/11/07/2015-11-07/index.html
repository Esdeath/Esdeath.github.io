使用storyboard或者xib自动布局应该注意的事项

發表於 2015-11-07

1.对于Label的处理

  1. label在storyboard中使用的时候,只需要设置位置的约束.
  2. Lines 设置为 0 就可以进行多行显示
  3. Label需要换行处理,这时候需要在初始化的时候设置 label的属性preferredMaxLayoutWidth.
    一般设置为[UIScreen mainScreen].bounds.size.width - 常数

2.关于Cell的高度计算问题

  1. 计算之前先要强制布局cell内部的所有子控件(label根据文字多少计算出自己最真实的尺寸)
    [self layoutIfNeeded]
  2. 在初始化的时候,设置tableView的估算高度.
    告诉tableView所有cell的估算高度(设置了估算高度，就可以减少tableView:heightForRowAtIndexPath:方法的调用次数)
    self.tableView.estimatedRowHeight = 200;

3.iOS的额外滚动区域处理

  1. 不需要自动增加scrollView的额外滚动区域
    self.automaticallyAdjustsScrollViewInsets = NO;

  2. 注意：在iOS7以后，导航控制器下的所有UIScrollView默认顶部都会添加额外的滚动区域（64）

  3. 在iOS 7中，苹果引入了一个新的属性，叫做[UIViewController setEdgesForExtendedLayout:]，它的默认值为UIRectEdgeAll。当你的容器是navigation controller时，默认的布局将从navigation bar的顶部开始。这就是为什么所有的UI元素都往上漂移了44pt。

4.移除容器视图中所有子控件

  1. 把容器视图的所有子控件全部移除父控件:

    [self.contentView.subviews makeObjectsPerformSelector:@selector(removeFromSuperview)];

5.设置tableView的额外滚动区域

`self.tableView.contentInset = UIEdgeInsetsMake(XMGHeadViewH + XMGTabViewH, 0, 0, 0);
